# Reports

This directory contains formal reports and analysis documents.

## Purpose

Store documentation for:

- Incident reports and post-mortems
- Root Cause Analysis (RCA) documents
- Performance reports
- Security audits
- Code review summaries

## Naming Convention

Use descriptive, kebab-case filenames with date prefix:
- `YYYY-MM-DD-incident-report-description.md`
- `YYYY-MM-DD-rca-issue-name.md`
- `performance-report-feature.md`
